---
title: Documentation Builder
description: Build and publish customer-facing documentation without leaving Orcha.
---

Most teams maintain docs in a separate tool with its own login, its own deploy pipeline, and its own set of problems. The documentation builder keeps everything in one place -- write, publish, and update help articles directly from Orcha.

## How it works

Create documentation pages from within your Orcha project using the built-in editor. When you publish, pages are deployed to S3 storage and become publicly accessible.

Published articles are automatically indexed by the support widget's search, so your users find relevant help before they submit a support request.

## Why this matters

Your docs and your tickets live in the same tool. When a support issue reveals a gap in documentation, you can write the article and close the ticket in the same session. No context switching, no second deploy pipeline, no stale docs rotting in a wiki nobody maintains.
